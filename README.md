# 테니스일지 PWA

주니어 엘리트 선수를 위한 테니스 훈련 일지. 설치형 웹앱(PWA)입니다.

## 폴더 구성
```
index.html        앱 본체 (단일 파일)
manifest.json     PWA 설정 (이름, 색상, 아이콘)
sw.js             서비스워커 (설치 + 오프라인 + 자동 업데이트)
icons/
  icon-192.png
  icon-512.png
  icon-maskable-512.png   (안드로이드 적응형 아이콘)
  apple-touch-icon.png    (iOS 홈 화면)
  favicon-32.png
```
이 5개 항목(파일/폴더)을 **같은 위치에 그대로** 올리면 됩니다.

## GitHub Pages 배포
1. GitHub에 새 저장소(repository)를 만들고 위 파일들을 그대로 push 합니다.
   (index.html, manifest.json, sw.js 가 저장소 최상단에 있어야 합니다.)
2. 저장소 → **Settings → Pages** 로 이동.
3. **Source** 를 `Deploy from a branch` → `main` 브랜치 / `/ (root)` 선택 후 저장.
4. 1~2분 뒤 `https://<사용자명>.github.io/<저장소명>/` 주소가 생성됩니다.

> 경로는 상대경로(`./`, `manifest.json`, `sw.js`)로 작성되어 있어, 저장소 하위 경로
> (`/저장소명/`)에서도 그대로 동작합니다. 별도 수정이 필요 없습니다.

## 크롬에서 설치 (안드로이드 / 데스크톱)
- 위 https 주소를 크롬으로 엽니다.
- 주소창 오른쪽의 **설치(⊕)** 아이콘을 누르거나, 메뉴(⋮) → **앱 설치** 를 선택합니다.
- 홈 화면 / 앱 목록에 "테니스일지" 아이콘이 추가됩니다.

## iOS(사파리)에서 설치
- 공유 버튼 → **홈 화면에 추가** 를 선택합니다.

## 데이터 보관과 백업 (중요)
모든 기록은 이 기기의 브라우저(또는 설치된 앱) 저장 공간에만 저장됩니다.
- 크롬에서 **"캐시된 이미지 및 파일"만** 지우는 것은 안전합니다 (기록이 지워지지 않습니다).
- 하지만 **"쿠키 및 기타 사이트 데이터"**까지 함께 지우거나, 안드로이드 설정 → 앱 정보에서
  앱의 **저장용량(데이터) 지우기**를 하면 기록이 모두 사라질 수 있습니다.
- 앱은 실행될 때마다 브라우저에 "영구 저장"을 요청해 자동 삭제 가능성을 낮추지만,
  사용자가 직접 데이터를 지우는 것까지 막을 수는 없습니다.
- 그래서 **설정 → 백업 파일 내보내기**로 주기적(예: 1~2주마다)으로 JSON 백업을 남겨두는 것을 권장합니다.
  프로필 화면에서 마지막 백업 날짜를 확인할 수 있고, 14일 이상 백업이 없으면 홈 화면에 알림이 표시됩니다.

## 주간 자동 백업 (매주 월요일 6시)
- 자동 백업이 켜져 있으면(기본값 ON), **매주 월요일 아침 6시가 지난 뒤 앱을 처음 여는 순간**
  백업 JSON 파일이 자동으로 다운로드됩니다. 앱을 켜둔 상태라면 6시가 되는 시점에 실행됩니다.
- 웹앱(PWA)은 네이티브 앱과 달리 꺼져 있는 동안 스스로 실행될 수 없어, "정확히 6시 정각"이 아니라
  "6시 이후 첫 실행 시점"에 밀린 백업을 처리하는 방식입니다.
- 프로필 → 데이터 → **자동 백업 (매주 월 6시)** 에서 끄고 켤 수 있습니다.
- 이미 그 주 월요일 6시 이후에 수동 백업을 했다면 자동 백업은 건너뜁니다 (중복 다운로드 방지).

## AI 자동 연동 (Gemini)
AI 코칭 분석과 영어 문장 만들기는 이제 **버튼 한 번으로 결과까지 자동 저장**됩니다.
1. [aistudio.google.com/apikey](https://aistudio.google.com/apikey)에서 구글 계정으로 무료 API 키를 발급받습니다 (카드 등록 불필요).
2. 앱의 **설정 → AI 설정**에 키를 붙여넣습니다.
3. 키는 이 기기 브라우저에만 저장되며, 백업 파일이나 GitHub 저장소에는 포함되지 않습니다.
   따라서 **기기마다(선수 폰, 보호자 폰) 한 번씩** 입력해야 합니다.

- 무료 사용량: Flash 모델 기준 분당 약 10회, 하루 수백 회 수준이라 개인 사용에는 충분합니다.
- 무료 티어에서는 입력 내용이 구글 모델 개선에 활용될 수 있습니다.
- 키를 입력하지 않았거나 오류가 나면, 기존의 **수동 방식(Claude 열기 → 복사 → 붙여넣기)**을 그대로 쓸 수 있습니다.

## 앱을 수정한 뒤 (중요)
새 기능이 기기에 바로 보이게 하려면, 코드를 고친 뒤 `sw.js` 맨 위의
```
const CACHE = 'tennis-journal-v1';
```
버전을 `v2`, `v3` … 처럼 올려서 함께 배포하세요. 그러면 기존 캐시를 비우고
최신 화면을 받아옵니다. (HTML은 네트워크 우선이라 온라인이면 보통 즉시 갱신됩니다.)
